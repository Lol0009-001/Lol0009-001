# Project Overview

## 項目名稱
Bus AI Assistant

## 產品定位
這是一個針對個人日常出行而設計的智能助手。它利用 GPS、地理圍欄、公共交通資料與自然語言理解能力，在適合的時間主動詢問使用者目的地，並提供可執行的巴士乘車建議與提醒。

## 使用情境
### 情境 1：離開學校
- 使用者進入學校外圍 geofence
- 助手發出通知："你現在想去哪裡？"
- 使用者回覆："返屋企"
- 系統查詢附近站點與班次
- 助手提供：最近站、下一班車、預計總時間、是否要立即出發

### 情境 2：市中心臨時出行
- 使用者在 downtown 區域停留
- 助手提示常用目的地：Home / School / Walmart / Custom
- 使用者選擇 Walmart
- 系統給出最近班次與轉車建議

### 情境 3：接近落車站
- 系統追蹤行程狀態
- 在接近目的地下車站時提醒："下一站準備落車"

## 產品核心能力
1. 位置感知
2. 區域觸發
3. AI 對話理解
4. 巴士班次查詢
5. 智能提醒

## 非目標（第一版不做）
- 複雜多城市支援
- 即時自動購票
- 完整地圖導航替代品
- 高階多模態語音代理

## 成功指標
- 使用者能在 3 次點擊或一次簡單對話內拿到可用乘車方案
- 通知不過度打擾
- 路線與候車建議具實際可用性
- 首版能穩定演示 geofence -> query -> route suggestion -> reminder 這條主流程

## 風險與難點
- iPhone 背景定位限制
- 不同城市交通資料格式不一
- 若只有靜態 GTFS，提醒準確度有限
- 通知頻率太高會造成疲勞

## 解法方向
- 優先使用 geofence 而非高頻 GPS 輪詢
- 先做單城市或單資料來源 MVP
- AI 僅負責理解與表達，不負責底層路線計算
- 用規則引擎處理提醒邏輯
